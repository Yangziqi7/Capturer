# Capturer 产品需求文档 PRD

## 1. 产品概述

Capturer 是一个“现实元素自动变贴图”的移动端工具。用户打开网页或小程序后，摄像头画面自动显示，系统持续识别画面中心最重要的主体，并将该主体提取为可爱的贴图素材。第一阶段目标不是做完整修图工具，而是验证“看到喜欢的东西，不需要复杂操作，自动抓出来变贴图”的核心体验。

当前开发优先级：先实现手机网页端测试原型，再迁移到微信小程序版本。

## 2. 背景与机会

参考视频中的核心吸引力来自三个点：

- 即时性：用户不需要拍照、裁剪、抠图、导出等复杂流程。
- 自动性：系统理解用户正在对准的中心主体，并主动生成可保留内容。
- 情绪价值：输出不是普通截图，而是更适合收藏、聊天、分享的贴图。

因此，产品第一阶段应避免过多商业 UI、会员入口、贴纸库、模板市场等干扰，先把“摄像头实时识别 + 自动提取贴图”做顺。

## 3. 产品目标

### 3.1 阶段目标

V0 网页原型目标：

- 用户打开页面后能看到摄像头画面。
- 用户授权摄像头后，不需要点击按钮即可进入自动识别。
- 系统优先识别画面中心最重要的主体。
- 系统自动生成一个贴图预览。
- 不调用 AI API，不消耗 token，不依赖后端。

V1 小程序 MVP 目标：

- 在微信小程序内复刻摄像头自动识别体验。
- 支持保存生成贴图到用户贴纸库。
- 提供基础分享链路。
- 后续再接入真实 AI 抠图或端侧模型。

### 3.2 成功指标

- 首次可见摄像头画面时间：小于 2 秒，权限等待不计入。
- 授权后自动识别启动时间：小于 1 秒。
- 中心主体框稳定时间：小于 3 秒。
- 贴图预览生成频率：主体稳定后 1-2 秒内自动更新。
- 用户无需阅读说明即可理解页面用途。

## 4. 用户与场景

### 4.1 目标用户

- 喜欢用表情包、贴纸、梗图聊天的年轻用户。
- 宠物主、亲子用户、情侣、学生群体。
- 想快速保存生活中可爱物件或表情的人。

### 4.2 核心场景

- 用户看到宠物、杯子、玩偶、手写字、朋友表情等内容。
- 打开 Capturer，对准主体。
- 系统自动框选画面中心主体。
- 右上角或底部浮层自动出现贴图结果。
- 用户后续可保存、分享、漫画化或加入贴纸包。

## 5. 当前版本范围

### 5.1 V0 网页原型必须实现

- 全屏摄像头画面。
- 首次进入自动请求摄像头权限。
- 无商业展示、无会员卡片、无额度、无贴纸库。
- 本地 Canvas 分析画面中心主体。
- 自动绘制识别框。
- 自动生成贴图预览。
- 本地轻量漫画化效果：提亮、增强对比、色阶压缩、白色贴纸描边。
- 摄像头不可用时显示模拟画面，方便继续调试识别逻辑。

### 5.2 V0 明确不做

- 不调用 OpenAI 或其他 AI API。
- 不做真正语义级抠图。
- 不做用户登录。
- 不做支付。
- 不做复杂贴纸库。
- 不做品牌模板或活动营销后台。

## 6. 核心交互

### 6.1 页面打开

系统行为：

- 页面加载后立即请求摄像头权限。
- 若用户授权，显示实时摄像头画面。
- 若用户拒绝或浏览器不支持摄像头，显示模拟画面和简短错误提示。

用户看到：

- 全屏摄像头或模拟画面。
- 一个极少干扰的状态条。
- 一个贴图预览浮层。

### 6.2 自动识别

系统行为：

- 每隔约 260ms 分析一帧低分辨率画面。
- 画面中心区域权重最高。
- 综合颜色差异、亮度差异、饱和度、局部边缘对比，估算主体区域。
- 对识别框做平滑处理，避免抖动。

用户看到：

- 绿色识别框自动围住中心主体。
- 状态条提示“中心主体已锁定”或“请把主体靠近中心”。

### 6.3 自动生成贴图

系统行为：

- 当主体框连续稳定后，自动裁切主体区域。
- 添加软边透明遮罩。
- 添加白色贴纸描边。
- 应用轻量漫画化滤镜。
- 更新贴图预览。

用户看到：

- 贴图浮层自动刷新，不需要点击。

## 7. 技术方案

### 7.1 当前纯前端方案

文件范围：

- `web/index.html`
- `web/styles.css`
- `web/app.js`

主要模块：

- Camera：使用 `navigator.mediaDevices.getUserMedia` 请求后置摄像头。
- Frame Renderer：把摄像头画面按 cover 方式绘制到低分辨率 Canvas。
- Subject Analyzer：按网格采样计算中心主体显著性。
- Overlay Renderer：绘制识别框和暗化遮罩。
- Sticker Renderer：裁切主体、添加软边、描边和漫画化滤镜。

### 7.2 算法说明

V0 不做真实 AI 语义分割，采用中心显著性启发式：

- 将画面切成 28 x 36 网格。
- 采样每个网格的平均 RGB 和亮度。
- 以画面边缘区域估算背景色。
- 计算每个网格与背景的颜色差、亮度差、饱和度和局部对比。
- 对靠近画面中心的网格加权。
- 将超过阈值的中心网格合并为主体框。
- 如果主体不明显，退化为中心区域框。

### 7.3 性能要求

- 不在全分辨率视频帧上做 `getImageData`。
- 分析 Canvas 固定低分辨率，避免手机卡顿。
- 识别循环不高于约 4 次/秒。
- 贴图生成不高于约 1 次/秒。

## 8. 后续路线

### V0.1 网页体验打磨

- 优化主体框稳定性。
- 增加“长按保存贴图”的输出图。
- 增加用户可选开关：原始贴图 / 漫画贴图。
- 适配 iOS Safari 摄像头限制。

### V1 微信小程序

- 迁移到小程序 camera 组件。
- 保留自动识别和自动生成体验。
- 加入本地贴纸库。
- 支持保存到相册和分享给好友。

### V2 智能提取升级

- 评估端侧模型，如 MediaPipe 或 TensorFlow.js。
- 对人物、宠物、手部、文字分别做更精准识别。
- 根据设备性能动态选择启发式算法或端侧模型。

### V3 商业化

- 会员高清导出。
- 主题贴纸包。
- 宠物、亲子、情侣、节日模板。
- B2B 活动贴纸机。

## 9. 验收标准

### V0 验收

- 打开 `web/index.html` 不出现空白页。
- 页面无商业卡片、无大段说明、无会员入口。
- 摄像头授权成功后显示实时画面。
- 摄像头不可用时显示模拟画面。
- 画面中心放置明显物体时，识别框能自动靠近主体。
- 主体稳定后，贴图预览自动更新。
- 断网状态下仍可运行。
- 不需要任何 API key。

### 风险

- 浏览器出于安全限制，手机摄像头通常需要 HTTPS；`file://` 或局域网 HTTP 可能无法授权摄像头。
- 本地启发式算法不等同于 AI 抠图，背景复杂或主体与背景颜色接近时效果会下降。
- 漫画化为本地滤镜，不具备生成式 AI 的风格理解能力。

# Capturer

AI 随手贴微信小程序 MVP。产品目标是让用户用手机摄像头捕捉现实里的宠物、表情、手写字和小物件，并快速生成适合微信聊天分享的可爱贴纸。

## 当前版本

- 原生微信小程序项目，可直接用微信开发者工具打开。
- 首页实现摄像头捕捉、模拟 AI 生成、三种贴纸风格、每日 5 张免费额度。
- 贴纸库实现本地保存、预览、分享入口和主题包付费引导。
- 会员页实现订阅和单次贴纸包的商业化入口说明。
- 手机网页端已接入浏览器端 TensorFlow.js Coco SSD + DeepLab 双模型管线：先在本机定位物体，再用语义分割约束边缘，最后用本地算法裁切、抠图、描边并生成贴图预览。
- 产品 PRD 见 `docs/PRD.md`。

## 本地预览

### 手机网页测试版

```bash
python3 -m http.server 4173
```

然后访问：

```text
http://localhost:4173/web/
```

如需用手机真机测试摄像头，请让手机和电脑在同一局域网，并访问电脑局域网 IP 对应的地址，例如：

```text
http://你的电脑IP:4173/web/
```

手机摄像头权限通常要求 HTTPS；如果用 `file://` 或局域网 HTTP 打开，浏览器可能拒绝摄像头，但页面会显示本地模拟画面用于验证自动识别与贴图预览。


### 手机端性能策略

网页原型会根据设备自动进入手机性能模式：摄像头预览直接使用浏览器原生 `<video>` 渲染，不再每帧重绘到 canvas；绿色取景框固定在画面中心，用户通过移动手机把目标放进框内。模型只在手机相对稳定时对绿框内快照做低频检测，且会降低摄像头采样分辨率、减少单次检测候选数量，并把贴图计算尺寸降到 224px。手机端不做边缘虚化/透明抠图，预览直接展示绿框内提取图像，避免分割和描边拖慢实时预览。

### GitHub Pages 手机真机测试

本仓库包含 GitHub Actions 自动部署配置：合并到 `main` / `master` / `work` 分支或手动运行 `Deploy mobile web prototype to GitHub Pages` workflow 后，会把 `web/` 目录发布到 GitHub Pages。

首次使用需要在 GitHub 仓库设置中打开：

1. 进入 `Settings` → `Pages`。
2. 在 `Build and deployment` 的 `Source` 中选择 `GitHub Actions`。
3. 推送本仓库后，进入 `Actions` 查看部署任务；部署完成后打开 `Deploy to GitHub Pages` 输出的 URL。
4. 用手机访问 HTTPS 地址（通常是 `https://<你的 GitHub 用户名>.github.io/Capturer/`），授权摄像头后即可测试浏览器端物体检测、语义分割和贴图预览。

### 微信小程序版本

1. 安装并打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择本仓库根目录。
4. AppID 可先使用测试号或在 `project.config.json` 中替换为正式小程序 AppID。
